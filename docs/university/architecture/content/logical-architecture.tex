\section{Логическая архитектура}
\headingtextsep

В разделе приводится логическое разбиение системы на основные прикладные модули и показываются интерфейсы их взаимодействия.

\textheadingsep
\subsection{Состав модулей системы}
\headingtextsep

На уровне логической архитектуры выделены четыре основных модуля: пользовательский интерфейс, прикладная бизнес-логика, сервис уведомлений и слой данных.

\textheadingsep
\subsection{Взаимодействие модулей}
\headingtextsep

Пользовательский интерфейс обращается к интерфейсу прикладных сервисов. Прикладная бизнес-логика координирует работу с задачами, документами и заметками, использует сервис уведомлений и обращается к слою данных через абстракции репозиториев.

\textheadingsep
\subsection{UML-диаграмма компонентов}
\headingtextsep

UML-диаграмма компонентов логической архитектуры приведена в приложении А. На ней показаны основные программные модули системы и интерфейсы взаимодействия между пользовательским интерфейсом, прикладной логикой, сервисом уведомлений и слоем данных.
